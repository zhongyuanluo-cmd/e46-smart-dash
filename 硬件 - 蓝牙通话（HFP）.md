
**AP6256 硬件支持，软件可以实现。** 但有个坑：

```
HFP (Hands-Free Profile) 链路：

  手机 ──BT HFP──► AP6256 ──► BlueZ + PulseAudio
                                      │
                        ┌─────────────┼─────────────┐
                        ▼             ▼             ▼
                    上行(麦→手机)  下行(手机→喇叭)  AEC回声消除
                        │             │             │
                    双麦→降噪→编码  DAC→功放→喇叭  关键！
```

| 环节 | 状态 | 说明 |
|------|:---:|------|
| HFP 协议栈 | ✅ BlueZ + oFono 支持 | Linux 成熟方案 |
| 宽频语音(WB) | ✅ mSBC 编解码 | 16kHz 采样，比窄带清晰 |
| **AEC 回声消除** | ⚠️ 最大坑 | 车内空间小、反射强，AEC 调不好对方听到自己回声 |

### AEC 推荐

```
推荐：SpeexDSP 或 WebRTC AECM（软件，免费）

  麦克风 ←── 你的声音
  喇叭   ──→ 对方的声音 ──→ 又被麦克风拾取（回声）
  
  需要 AEC 算法实时抵消这个回声路径。
  WebRTC AECM 在 ARM A55 上跑占 CPU ~8-12%，效果很好。
```

> **结论：可以实现，但需要花精力调 AEC。** 如果不做免提通话，这个可以先放一放。
