
**赛道分析技术方案**：

> **架构结论 (2026-05-29)**：RK3566 NPU (0.8 TOPS) 无法本地运行可用 LLM。赛道模式的"理解"和"对话"层必须走云端。车机负责传感器采集 + 数据分析 + 规则引擎 + 模板播报。完整 CoDriver 体验（自然语言对话、赛后 AI 复盘）在手机端实现——高端手机 NPU (15-45 TOPS) 可本地运行 3B 参数赛道领域微调 LLM。
>
> **车机版 CoDriver = 传感器 + 分析 + 模板播报 + 云端 LLM，不需换芯片。**
> **手机版 CoDriver = 全部能力本地化，完整 CoDriver 体验。**
>
> 详见 `赛道助手CoDriver/openspec/` 项目文档。

数据采集流水线：
```
GPS (ATGM336H 10Hz → NEO-M9N 25Hz Phase 6) ──┐
                 ├── 10Hz→25Hz 同步 ──→ TrackLogger ──→ 圈分段器
IMU (ICM-42688 200Hz) ────────────────────────┘                              │
                                                 ▼
CAN (油门/刹车/车速) ──→ can-gateway ──→    LapCompare Engine
                                                 │
                                     ┌───────────┼───────────┐
                                     ▼           ▼           ▼
                                 制动分析    弯道分析     圈时差分析
                                     │           │           │
                                     └───────────┴───────────┘
                                                 ▼
                                           TTS 播报引擎
```

1. **弯道检测**：滑动窗口曲率法（Menger Curvature），自动识别弯道起始/顶点。首圈一次性检测，后续复用分段。

2. **制动点对比**——检测逻辑：
```
CAN 刹车压力 > 阈值 (如 2 bar) 且 车速在下降 → 标记为制动开始
                              ↓
                    记录此时 GPS 坐标 + 距下一个弯道的距离
                              ↓
                    与上一圈同一弯道的 BrakeEvent 对比
```
对比维度表格：
| 指标 | 数据来源 | 对比方式 |
|------|----------|----------|
| 刹车点偏移 | GPS 坐标 → 沿轨迹距离差 | Δdist = lap_N - lap_N-1 |
| 力度变化 | CAN 制动主缸压力 | Δpct = (max_p_N - max_p_N-1) / max_p_N-1 × 100% |
| 入弯速度 | CAN 车速 | 绝对值对比 |
| 制动稳定性 | IMU 纵向 G 标准差 | 衡量平顺度 |

3. **弯速对比**——速度曲线图：
```
速度
(km/h)
130 ┤                    Lap N-1 (上一圈)
    │        ╭──╮         Lap N   (当前圈)
110 ┤       ╱    ╲___     
    │      ╱         ╲___
 90 ┤     ╱              ╲___
    │    ╱                    ╲___
 70 ┤   ╱                         ╲___
    │──╱                              ╲___
    └──┬──────┬──────┬──────┬──────┬──────→ 距离
      入弯    弯心                          出弯
       ↑                        ↑
    晚制动                    早给油
    入弯慢                    出弯快
```

4. **圈时差归因**——归因流水线：
```
当前圈 vs 参考圈（最快圈/上一圈）
         │
         ▼
  按弯道分段计算 Δt
         │
  ┌──────┴──────┐
  ▼             ▼
Δt > 阈值       Δt ≈ 0
  │              → 跳过
  ▼
  对该弯道做子指标分解：
  ┌───────────┬───────────┬───────────┬───────────┐
  制动距离    入弯速度    弯心速度    出弯速度    给油时机
         │
         ▼
  与参考圈的差异向量 [Δbrake, Δentry_v, Δapex_v, Δexit_v, Δthrottle_timing]
         │
         ▼
  规则引擎 → 推断根因
```

根因推断规则：
| 特征组合 | 推断结论 | 置信度 |
|----------|----------|--------|
| 刹车点早 + 入弯速度低 + 弯心速度低 | 入弯太早 | 高 |
| 刹车点晚 + 入弯速度高 + 弯心速度低 + 侧向G低 | 入弯太快推头 | 高 |
| 刹车正常 + 入弯正常 + 弯心正常 + 出弯速度低 | 出弯给油晚了 | 中 |
| 全段速度都低 + 侧向G低 | 保守驾驶 | 低 |
| 刹车压力波动大 + 纵向G不稳 | 制动操作不稳 | 中 |

6. **CPU 负载**：实时 <3%，峰值 ~12%，完全可行。

**第二部分：核心算法代码骨架**（7 个模块，完整 Python 可运行代码）

1. 数据结构定义（dataclass：GPSPoint/IMUPoint/CANFrame/TrackSegment/BrakeEvent/CornerMetrics/LapRecord）
2. 弯道检测（Menger Curvature，滑动窗口，自动区分左右弯，直道填充）
3. 制动事件检测（CAN 压力阈值 + 车速下降，关联到前方弯道）
4. 弯道指标计算（min_speed/max_lateral_g/corner_duration/entry_exit_speed）
5. 圈时差归因分析（6 维特征向量，5 条根因规则库）
6. TTS 播报生成（自然语言: "T1 慢了 0.3 秒，入弯太早"）
7. 条件任务调度器（定时/阈值/圈事件触发，停止条件检查）

单圈数据量 ~86KB，所有算法在 RK3566 上实时可行。
