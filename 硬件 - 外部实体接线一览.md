> 最后更新: 2026-05-27

## 1. 主控 - IKE X11175（仪表盘后方 26 针黑色插头）

连接仪表 IKE X11175 接口，获取电源、CAN 总线、K-Bus。

| 信号分类     | X11175 引脚                      | 车载电压/电平          | 接入主控方式                                                                               |
| -------- | ------------------------------ | ---------------- | ------------------------------------------------------------------------------------ |
| 主电源      | Pin 5 (KL15)                   | 12V 钥匙电          | 进宽压 DCDC (MP4580) → 5V/3.3V 系统供电                                                     |
| 常电       | Pin 4 (KL30)                   | +12V 常电 (F43 5A) | 进 DCDC 的备用电源输入，用于 Suspend-to-idle 维持供电 + 电瓶电压 ADC 监控                                 |
| 主接地      | Pin 1 (GND)                    | 0V 参照地           | 主控板、降压板、所有收发器共地                                                                      |
| 动力 CAN   | Pin 9 (CAN_H) / Pin 10 (CAN_L) | ~2.5V 差分         | 双绞 50cm → MCP2515-E (控制器) + TJA1050/SN65HVD230 (收发器) → RK3566 SPI1 (Pin 19/21/23/24) |
| 车身 K-Bus | Pin 14 (K-Bus)                 | 12V 单线           | 单线 50cm（与 GND 绞合）→ TH3122.4 K-Bus 收发器 → RK3566 UART4 (ttyS4, Pin 32/33)              |
|          |                                |                  |                                                                                      |

### 1.1 电源策略说明

```
KL15 ON (开车):
  KL15 → DCDC → RK3566 全速运行 + 所有外设正常供电
  功耗: ~5W (CarPlay) / ~3W (纯UI)

KL15 OFF, < 16 分钟 (短暂停车):
  KL30 常电维持 → RK3566 Suspend-to-idle (冻结用户空间)
  CAN 保持监控作为唤醒源
  功耗: ~0.3W (50mA@5V)
  恢复: < 1 秒

KL15 OFF, > 16 分钟 (过夜):
  KL30 也切断 → 完全断电
  下次 KL15 ON → 冷启动 5-8 秒
```

### 1.2 K-Bus 收发器选型

采用 **Melexis TH3122.4**（SOIC-16），与 BlueBus / Resler Interface 同款：
- 硬件碰撞检测（K-Bus 是多主总线，必须）
- 睡眠/唤醒功能
- 内置 5V 稳压输出
- 车规温度范围

> ~~MCP2003~~（LIN 收发器，不适合 K-Bus 多主架构）
> ~~L9637D~~（ISO 9141 K-Line 诊断收发器，无碰撞检测）

### 1.3 CAN 终端电阻注意事项

E46 原车 PT-CAN 两端已有 120Ω 终端电阻（DME + DSC 内部）。
MCP2515-E 模块上的 120Ω 电阻 **必须拆除**（通常标号 R2 或 R120），否则全车总线阻抗从 60Ω 降至 40Ω，信号瘫痪。

## 2. 音频输出

### 2.1 Business CD 车型（直连）

```
RK3566 I2S0 → PCM5102A DAC → 模拟音频 → Business CD 17-pin 圆形连接器
                                              ├── Pin 4: AUX-L
                                              ├── Pin 5: AUX-R
                                              ├── Pin 8: AUX-GND
                                              └── Pin 6: AUX-EN (接 GND 激活 AUX)
```

条件：CD53 固件 ≥ v42，AUX 功能需 NCS Expert 开启。

### 2.2 Board Monitor 车型（WiFi 音频桥）

Board Monitor 车型的音频输入在后备箱 BM54 模块，距离中控台 ~4m，不走长模拟线。

```
中控台 (RK3566)                              后备箱
┌─────────────────────┐                    ┌──────────────────┐
│ PulseAudio 混音      │   WiFi UDP/RTP     │  ESP32-S3        │
│ (TTS + A2DP + 提示)  │ ─── PCM 音频流 ──► │  ├── WiFi 接收    │
│ RTP 发送             │                    │  ├── I2S 输出     │
└─────────────────────┘                    │  └── Ring Buffer  │
                                           │         │         │
                                           │         ▼         │
                                           │    PCM5102A DAC   │
                                           │         │         │
                                           │   模拟音频 → BM54 │
                                           └──────────────────┘
```

- 协议: RTP over UDP, PCM 16bit 48kHz stereo
- 带宽: ~192KB/s
- 延迟: 20-30ms (固定 jitter buffer)
- RK3566 开 WiFi AP (hostapd)，ESP32-S3 自动连接
- ESP32-S3 只需 ~200 行固件代码

## 3. 其他外设

### 3.1 GPS 天线
ATGM336H / NEO-M9N (Phase 6) 模块的有源陶瓷天线，贴于仪表台下方塑料盖板内侧，陶瓷面朝上、底部垫金属反射面（铝箔）。两个模块共用同一天线（IPEX 接口）。

### 3.2 胎压天线
CC1101 模块外接 315MHz 弹簧天线，优化 RF 信号接收。

### 3.3 系统提示音喇叭
从 PCM5102A DAC 或独立 PWM 音频输出，接一个小型扬声器（如 MAX98357A I2S 功放 + 3W 喇叭），专门用于系统报警音（与音乐通道独立，确保报警不被音乐音量影响）。

### 3.4 麦克风
双 MEMS 麦克风 (ICS-43434 ×2) → PDM → RK3566 内置 PDM 接口 → 软件波束成形，安装于 IKE 框架上，正对主驾方向。使用屏蔽双绞线，单点接地。
