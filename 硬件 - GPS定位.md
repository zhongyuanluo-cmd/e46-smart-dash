## 为什么需要 25Hz？

200 km/h 下的 GPS 采样间距：

| 刷新率 | 采样间距 | 直观感受 |
|:---:|:---:|------|
| 1Hz | 55.6m | 十几个车身 |
| 10Hz（ATGM336H） | 5.56m | 超过一个 E46 车长 (4.49m) |
| **25Hz（NEO-M9N）** | **2.22m** | **半个车身** |
| 100Hz | 0.56m | 引擎盖一半 |

> 25Hz + 200Hz IMU 融合：GPS 每 8 帧 IMU 校准一次，航位推算误差极小。等效定位精度 ~0.3m。

### NEO-M9N 是单频还是多频？

NEO-M9N 是**单频（L1 only）模块**，不支持 L2/L5 双频。

之所以 25Hz + 单频方案在这个项目里够用，是因为：

1. **车机有 CAN 总线**：可以直接读取轮速传感器的车速（误差 <1%），GPS 的主要任务是**定位**（知道在哪条赛道的哪个位置），不是测速
2. **IMU 融合**：NEO-M9N 的 1.5m 单点精度 + 200Hz IMU + Kalman Filter 融合后，有效定位精度可达 ~0.3-0.5m（GPS 两点之间 IMU 高频插值）
3. **成本约束**：双频模块（如 u-blox F9 系列）价格高 5-10 倍，而通过 CAN + IMU 融合也能逼近同等水平

#### 对比 Garmin Catalyst

Garmin Catalyst 用的是多频 GNSS（L1+L5），配合高精度陶瓷天线 + IMU 融合 + 预装赛道地图匹配，定位精度 ~0.3m。但它只有 10Hz 输出频率——Garmin 不靠采样率，靠的是单点精度。

NEO-M9N 换了策略：用 **25Hz 高采样率** 弥补单频精度的不足。在无 CAN 数据时（如 CoDriver 手机 App 场景），更高的采样率 + Kalman Filter 是提升有效定位精度最直接的手段。

## 硬件接线

```
NEO-M9N / ATGM336H 模块:
  VCC   ───  3.3V
  GND   ───  GND
  TXD   ───  RK3566 UART1_RXD
  RXD   ───  RK3566 UART1_TXD
  PPS   ───  可选，GPIO 时间同步

天线:
  IPEX 接口 ─── 有源陶瓷天线 (25×25mm, 3.3V 馈电)
```

天线安装位置：仪表台下方塑料盖板内侧，陶瓷面朝上，底部垫金属反射面（铝箔）。

## 软件配置

### ATGM336H（Phase 1-5）
```bash
# /boot/config.txt
# UART1 使能
dtoverlay=uart1

# 应用层：stty 9600 8N1，NMEA 0183
```

### NEO-M9N（Phase 6 升级）
```bash
# 波特率升级到 115200（25Hz NMEA 需要带宽）
# 模块默认 38400，用 u-center 改为 115200 并保存
# NMEA 消息精简：只保留 RMC + GGA（节省带宽）
```

用 u-center 配置 NEO-M9N：
1. 连接 USB-UART → 模块
2. View → Messages View → UBX-CFG-RATE → 250ms (25Hz)
3. UBX-CFG-PRT → UART1 Baudrate → 115200
4. UBX-CFG-MSG → 关闭不需要的 NMEA 消息（GSV, GSA, VTG, GLL）
5. UBX-CFG-CFG → Save（保存到 Flash）
