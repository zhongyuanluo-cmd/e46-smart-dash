

  
最终方案：

- 方向盘语音键 → 永远车机小马（GPIO → Core3566）
- "小马小马" → 永远车机 NLU（NPU 常开）
- "Hey Siri" → 永远 Siri（iPhone AOP + 车机麦克风通道 + 手机麦克风辅助）

零模式切换，零状态机，零混淆。Siri 本来就支持全语音唤醒，不需要占用方向盘按键。


两种状态差异对照表：  

| 路径                 | CarPlay 未连接                  | CarPlay 已连接                           |  
|-----------------|-----------------------------|-----------------------------------|  
| 音乐音频          | BlueZ A2DP iPhone→车机 | CarPlay 原生音频 iPhone→车机 |  
| 通话麦克风       | HFP 通道车机→iPhone      | CarPlay 通道车机→iPhone         |  
| 方向盘语音键    | → 车机小马                       | → 车机小马（不变）                   |  
| "小马小马"唤醒 | → 车机 NPU                     | → 车机 NPU（不变）                  |  
| "Hey Siri"唤醒  | iPhone 自带麦克风            | iPhone AOP 双通道                    |  
| Siri 通话/指令   | 通过蓝牙 HFP                    | 通过 CarPlay 协议                       |  
| 车机 TTS 播报  | 独占 DAC → 扬声器           | 与 CarPlay 音频混音 → DAC        |

核心结论：方向盘按键和"小马小马"在 CarPlay 连接前后行为完全一致，车主无需感知状态切换。


**语音交互架构最终方案**：

```
┌─────────────────┼─────────────────┐
│                 │                 │
▼                 ▼                 ▼
"小马小马"       方向盘语音键      "Hey Siri"
│                 │                 │
▼                 ▼                 ▼
车机 NPU         车机 NPU        iPhone AOP
(常开唤醒词)     (物理唤醒)      (全时监听)
│                 │                 │
▼                 ▼                 ▼
车机 NLU         车机 NLU          Siri
(车辆/赛道/空调)  (车辆/赛道/空调)  (通话/短信/通用)
```

- "小马小马" + 方向盘语音键 → 车机（PulseAudio 分流，不受 CarPlay 影响）
- "Hey Siri" → iPhone AOP 全时监听，含手机麦克风辅助
- 零模式切换，零状态机，零混淆
