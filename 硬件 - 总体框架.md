
这是一套安装在宝马 E46车型上的 智能车机

- 总控：Core3566 + AP6256 WiFi/BT（板载）
- 显示：6.9" MIPI DSI 异形屏
- 总线：PT-CAN (MCP2515-E + TJA1050)、K-Bus (TH3122.4/UART)、ADC 电压监控
- 传感器：GPS (ATGM336H 10Hz → NEO-M9N 25Hz Phase 6)、IMU ICM-42688-P、TPMS CC1101
- 音频：双 MEMS (PDM) → RK3566 PDM 接口 → 软件波束成形 → PCM5102A DAC
- 语音：NPU 跑 ASR + CPU Piper TTS
- 双音频架构：Business CD 直连 / Board Monitor 走 ESP32-S3 WiFi 音频桥
- 无线 CarPlay：5GHz 连 iPhone + 2.4GHz 音频桥，双频隔离
- 软件栈：Armbian → Qt6/QML + Mali-G52 + SocketCAN + PulseAudio + DBus
- 外部存储：TF或micro SD卡槽



| 子系统     | 推荐方案                 |   外挂/主控    | 理由                           |
| ------- | -------------------- | :--------: | ---------------------------- |
| PT-CAN  | MCP2515-E + TJA1050 | 外挂 SPI | MCP2515 控制器(E-temp 125°C) + TJA1050 收发器(车规 150°C) |
| K/I-Bus | TH3122.4             |  外挂 UART   | 硬件碰撞检测，BlueBus同款             |
| GPS     | ATGM336H → NEO-M9N (Phase 6) |  外挂 UART   | Phase 1-5 用 ATGM336H 10Hz, Phase 6 升级 25Hz |
| IMU     | ICM-42688-P          |   外挂 SPI   | 车规温度范围                       |
| TPMS    | CC1101               |   外挂 SPI   | 315MHz 万能芯片                  |
| 蓝牙音频    | AP6256               |   **主控**   | Core3566 板载，A2DP 不费 CPU      |
| DAC     | PCM5102A             |   外挂 I2S   | ¥3，音质秒杀板载                    |
| 双麦阵列    | ICS-43434×2           | 外挂 PDM | RK3566 原生 PDM 接口，无需外置 ADC     |
| 语音识别    | RK3566 NPU + CPU     |   **主控**   | NPU 卸声学模型，CPU 做 NLU          |
| TTS     | Piper TTS            |   **主控**   | CPU 15%，免费，音质够用              |
| 电压监控    | ADC 分压               |   **主控**   | 两个电阻的事                       |

### 赛道模式 CPU 负载预估

| 任务 | CPU 核心 | 负载 | 
|------|:---:|:---:|
| GPS 数据采集 | 核0 | 1% |
| IMU 200Hz | 核0 | 3% |
| CAN 50Hz | 核0 | 2% |
| TPMS 接收 | 核0 | 1% |
| 数据记录(写 eMMC) | 核0 | 5% |
| UI 渲染(60fps) | 核1+GPU | 15% |
| **ASR 声学模型** | **NPU** | **0% CPU** |
| TTS 合成 | 核2 | 15% |
| NLU 推理 | 核2 | 5% |
| PulseAudio 混音 | 核3 | 2% |
| BlueZ 蓝牙 | 核3 | 3% |
| **总 CPU** | | **~50%** |

> 赛道并发模式下 CPU 总负载约 50%，NPU 独立处理语音推理，GPU 独立渲染——**发热可控，流畅无压力。**




Micro SD 插槽：是否需要？

**需要，但不是给系统用，是专门给赛道遥测日志用。**

```
存储架构：

  eMMC 32GB (板载焊接)
  ├── Linux 系统 + 应用           ~4GB
  ├── ASR/TTS/NLU 模型           ~0.5GB
  ├── UI 资源 + 字体             ~0.5GB
  ├── CAN 日志缓冲区              ~1GB
  └── 剩余                        ~26GB

  Micro SD (可插拔，128GB+)
  └── 赛道遥测数据 (CSV/二进制)
      ├── 每条赛道记录 50-200MB
      ├── 可插拔 → 拿回家分析
      └── 换卡 = 无限容量
```

### 赛道数据量计算

| 数据源 | 速率 | 每小时 |
|--------|------|:---:|
| GPS (10Hz → 25Hz NMEA) | 1KB/s → 2.5KB/s | 3.6MB → 9MB |
| IMU (200Hz 6轴) | 4.8KB/s | 17MB |
| PT-CAN (50帧/s) | 0.8KB/s | 2.9MB |
| **合计** | **6.6KB/s** | **~24MB** |

一条 20 分钟的赛道 session ≈ 8MB，一张 32GB 卡存 4000 条。**128GB 卡只要 ¥30，够你跑一辈子赛道。**

### 更重要的是"拔卡导出"

```
赛道跑完 → 熄火 → 拔卡 → 插电脑 → 数据分析 (MoTec / Circuit Tools)
```

这个工作流是赛道玩家的核心需求，SD 卡的物理可插拔性无可替代。
