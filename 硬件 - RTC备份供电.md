# 硬件 - RTC 备份供电

> 最后更新: 2026-05-27  
> 状态: ✅ 方案确定，待采购验证

---

## 背景

RK3566 SoC 内置 RTC（集成在 PMIC RK809-5 中），但 Core3566 核心板和 CM4-IO-BASE 底板均无备份电池电路。每次完全断电（KL30 断开），系统时间丢失。

**解决方案**：超级电容方案 — 车规温度、免维护、10 天断电保持。

---

## 电路设计

```
                   系统 3.3V
                       │
                      ┌┴┐
                      │ │ 100Ω（限流电阻）
                      └┬┘
                       │
                       ├──── 二极管 1N4148（阳极）
                       │       │
                       │       ▼（阴极 → RTC）
                       │       │
               ┌───────┴───────┼──────────────► CM4 Pin 99 (VDD_RTC)
               │               │
              ┌┴┐             ┌┴┐
         2.5F ===              │ 0.1μF（去耦）
         5.5V  │              └┬┘
               │               │
              GND             GND


原理：

  开车时（3.3V 有电）：
    3.3V → 100Ω → 二极管 → RTC 供电 + 超级电容充电
    充电时间: ~15 秒充满（限流 100Ω, 峰值电流 ~33mA）

  熄火断电（3.3V 丢失）：
    超级电容 → RTC（二极管反向截止，不会倒灌系统电路）
    维持时间: ~10 天（RTC 功耗 ~3-5μA, 2.5F 从 3.1V 放电到 1.5V）
```

## 放电时间计算

```
RK809-5 PMIC RTC 模块：
  待机电流 I_rtc ≈ 3-5 μA（典型值）

超级电容：
  电容 C = 2.5F
  充满电压 V_max = 3.1V（3.3V - 二极管压降 0.2V）
  最低保持 V_min = 1.5V

可用电量:
  Q = C × ΔV = 2.5F × (3.1V - 1.5V) = 4.0 库仑

放电时间:
  T = Q / I_rtc
  = 4.0C / 3μA ≈ 1,333,333 秒 ≈ 370 小时 ≈ 15.4 天
  = 4.0C / 5μA ≈ 800,000 秒   ≈ 222 小时 ≈  9.3 天

  → 保守估计: ~10 天
  → 充足余量: 日常停车场景完全覆盖
```

## 备选：配合 GPS 自动授时

```
开机流程:
  │
  ├── RTC 有电（断电 < 10 天）
  │   → 系统时间正确 → 无需 GPS 授时 → 立即开机
  │
  └── RTC 掉电（断电 > 10 天）
      → 系统时间 1970-01-01
      → GPS 冷启动获取 UTC (~30s)
      → systemd-timesyncd 自动同步 → 时间恢复
```

> 即使 RTC 掉电，GPS 冷启动 30 秒对日常用车体验影响可忽略。

## PCB 布局注意

```
1. 超级电容远离高温源（SoC、电源模块）
   → 中控台底部靠近外壳通风处优先

2. 二极管靠近 RTC 引脚放置
   → 减少反向漏电路径

3. 限流电阻功率校核
   → P = I²R = (33mA)² × 100Ω = 0.11W
   → 0805 封装额定 0.125W，刚好够，建议用 1206 (0.25W)

4. 超级电容焊接注意极性
   → 反接会直接爆浆，丝印务必标注 + / -
```
