## 一、IKE X11175 → 主控板 连接方案

### 分两组插头：电源 + 信号

```
         IKE X11175 端                              主控板端
    ┌─────────────────────┐                  ┌─────────────────────┐
    │ Pin 5  KL15  ═══ 红 │                  │ Micro-Fit 3.0 2P   │
    │ Pin 1  GND   ═══ 黑 │─── 电源组 ───→  │ ├─ Pin1: KL15 (12V) │
    └─────────────────────┘                  │ └─ Pin2: GND        │
                                             └─────────────────────┘
    ┌─────────────────────┐                  ┌─────────────────────┐
    │ Pin 9  CAN_H ═══ 绿 │                  │ JST SM 2.5mm 3P    │
    │ Pin 10 CAN_L ═══ 白 │─── 信号组 ───→  │ ├─ Pin1: CAN_H      │
    │ Pin 14 K-Bus ═══ 黄 │                  │ ├─ Pin2: CAN_L      │
    └─────────────────────┘                  │ └─ Pin3: K-Bus      │
                                             └─────────────────────┘
```

### 选型理由

| 插头 | 用在哪 | 针数 | 单针电流 | 选型理由 |
|------|--------|:---:|:---:|------|
| **Micro-Fit 3.0** | 电源组 (KL15 + GND) | 2P | 5A | 启动瞬态 2A，需余量；带锁扣防震 |
| **JST SM 2.5mm** | 信号组 (CAN_H/L + K-Bus) | 3P | 3A | 体积小，CAN/K-Bus <100mA 绰绰有余 |

### 为什么不合并成一个插头？

1. **防呆**：电源和信号插头大小完全不同，盲插不会反
2. **排查方便**：CAN 通信异常时可单独拔信号插头测试，不用断电
3. **电流匹配**：不用为 3 根弱信号线采购 5A 的大插头

| 物料 | 数量 | 关键词 |
|------|:---:|------|
| Micro-Fit 3.0 2P 公母套 | 2 | "Micro-Fit 3.0 2P 双排" |
| Micro-Fit 3.0 母端子 | 4 | "Micro-Fit 3.0 母端子 20-24AWG" |

## 二、IKE 侧（车上）

IKE 插头是 BMW 原厂 **X11175**（26 针黑色），不做替换，采用 **并线引出** 方式：

- IKE 插头不剪断，用 **T 型分线夹**（ScotchLok）或 **破线并焊** 从原车线束引出
- 引出端接上插头母端（装在主控板上）
- 主控板端用公端

### 使用引脚

| X11175 Pin | 信号 | 线色（参考） | 备注 |
|:---:|------|------|------|
| 1 | GND | 棕 | 主接地 |
| 5 | KL15 | 紫/灰 | 钥匙 ON = 12V |
| 9 | CAN_H | 黄/红 | PT-CAN 高 |
| 10 | CAN_L | 黄/棕 | PT-CAN 低 |
| 14 | K-Bus | 白/红/黄 | 车身总线 |

## 五、CAN 终端电阻

> ⚠️ **关键**：上车前必须检查

| 位置 | 阻值 | 操作 |
|------|------|------|
| MCP2515 模块（PCB 上） | 120Ω | **必须拆除**（标号 R2 或 R120） |
| E46 原车 DME 内部 | 120Ω | 保留（原车已有） |
| E46 原车 DSC 内部 | 120Ω | 保留（原车已有） |

原车 PT-CAN 两端各有一个 120Ω 终端电阻（DME + DSC），并联阻抗 = 60Ω。
再加一个 120Ω（MCP2515 模块）会使阻抗降到 40Ω，信号衰减严重。
**拆除模块上的 120Ω 是关键步骤，不要遗漏。**

## 六、GPS 天线 + 双阵列麦克风

### 6.1 GPS 模块与天线

```
                 ┌──────────────────┐
  GPS 有源陶瓷   │  IPEX 1代        │  UART
  天线 ─────────┤→ ATGM336H /     ──┤──→ Core3566 UART1
  (25×25mm)     │  NEO-M9N (Phase6) │  (Pin 8/10, ttyS1)
                 └──────────────────┘
```

| 连接点 | 接口类型 | 备注 |
|--------|---------|------|
| 天线 → GPS 模块 | **IPEX 1代**（模块自带底座） | 有源陶瓷天线，3.3V 馈电由模块提供 |
| GPS 模块 → Core3566 | **JST SM 2.5mm 2P** 或直接焊排针 | UART TX/RX，很短（模块就在主板上） |
| 天线安装位置 | 仪表台下方塑料盖板内侧 | 陶瓷面朝上，底部垫铝箔反射面 |

> GPS 天线用 IPEX 非常方便——插拔不需要工具，体积小。不要用 SMA，那是外置天线才需要的。

### 6.2 双阵列 MEMS 麦克风（Phase 5）

```
  ┌─────────────┐       ┌──────────────┐
  │ ICS-43434   │       │ ICS-43434    │
  │ (麦克风 #1) │       │ (麦克风 #2)  │
  │ 左 A 柱     │       │ 右 A 柱      │
  └──┬──┬──┬───┘       └──┬──┬──┬─────┘
     │  │  │              │  │  │
     │  │  └─ DATA ───────┼──┼──→ Core3566 PDM 输入
     │  └─── CLK ─────────┤  │
     └───── VDD/GND       │  │
                          └──┘
                    (两片麦克风共享 CLK 线)
```

| 连接点 | 接口类型 | 针数 | 备注 |
|--------|---------|:---:|------|
| 麦克风 → 主板 | **JST SH 1.0mm 4P** | 4 | VDD / GND / CLK / DATA |
| 屏蔽 | 双芯屏蔽线 | — | 信号线外包铜箔 + 接地，50cm 走模拟会引入噪声 |

**重要**：RK3566 只有 **1 路 PDM** 输入（立体声模式可接 2 片 ICS-43434，左右声道共享 CLK）。两片麦克风的 DATA 线分别接 RK3566 PDM_SDI0 和 PDM_SDI1。

**淘宝关键词**：
| 物料 | 关键词 |
|------|--------|
| ICS-43434 模块 | "ICS-43434 I2S MEMS 麦克风模块" |
| 麦克风插头 | "JST SH 1.0 4P 公母" |

## 七、散热风扇 + 温度传感器

### 7.1 散热风扇

CM4-IO-BASE-B 底板自带专用风扇接口：

```
  底板 FAN 接口（4-pin 2.54mm 排针）
  ┌──────────────────────────┐
  │ Pin1: GND                │
  │ Pin2: 5V                 │
  │ Pin3: TACH (转速反馈)    │
  │ Pin4: PWM (调速)         │
  └──────────────────────────┘
        ↓
  标准 4-pin PWM 风扇
  (5V, 30×30mm 或 40×40mm)
```

| 参数 | 要求 |
|------|------|
| 电压 | **5V**（不是 12V！芯片是 EMC2301） |
| 尺寸 | 30×30×10mm 或 40×40×10mm |
| 接口 | 标准 4-pin PWM 风扇插头（2.54mm 间距） |
| 控制 | EMC2301 I2C 风扇控制器（内核自带驱动） |

> ⚠️ **必须先接风扇再通电**——EMC2301 芯片空载上电会烧毁。
> 底板 I2C-10 共享：FAN (0x2f) + RTC (0x51)，不冲突。

淘宝搜 "5V 30mm PWM 风扇 4pin"，¥5-15。

### 7.2 屏幕背后温度传感器

贴在 ER-TFT069-1 屏幕背面，监控屏温（上限 70°C）。

| 方案 | 芯片 | 接口 | 插头 | 优点 |
|------|------|------|------|------|
| **推荐** | DS18B20 | 1-Wire | JST SM 2P | 防水探头、一根线挂多个、驱动成熟 |
| 备选 | LM75 / TMP102 | I2C | JST SH 4P | 更精确、可挂在已有 I2C 总线上 |

推荐 **DS18B20**——淘宝有 TO-92 封装 + 引线的探头版本（¥5），1-Wire 只需要 GND + DQ 两根线到 Core3566 任意 GPIO，最省插头。

```
  DS18B20 (TO-92 封装, 贴在屏后)
       │
       ├─ VDD ─── 3.3V
       ├─ DQ  ─── Core3566 GPIO (如 Pin 11/13)
       ├─ GND ─── GND
       │
       └─ JST SM 2P (GND + DQ，VDD 可主板端就近取)
```

**淘宝关键词**：
| 物料 | 关键词 |
|------|--------|
| 5V PWM 风扇 | "5V 3010 PWM 风扇 静音" |
| DS18B20 探头 | "DS18B20 防水探头 温度传感器" |
| 1-Wire 插头 | "JST SM 2P" |

## 八、胎压传感器（TPMS）

### 信号接收

| TPMS 类型 | 接收方式 | 频率 | 是否需要天线/插头 |
|-----------|---------|------|:---:|
| **BLE TPMS**（后装） | AP6256 蓝牙 BLE 扫描 | 2.4GHz | ❌ 不需要，板上已有 |
| **315MHz TPMS**（原厂） | CC1101 SPI 模块 | 315MHz | ✅ 需要弹簧天线 + SPI 接线 |

### BLE TPMS（推荐后装）

```
  胎压传感器 (4个轮胎)              主控
  ┌─────────────────┐         ┌──────────────┐
  │ BLE advertising │   ═══   │ AP6256 BLE   │
  │ 持续广播胎压数据 │  无线   │ 扫描接收      │
  └─────────────────┘         └──────────────┘
```

**零额外接线**，AP6256 蓝牙 BLE 直接接收。A2DP 音乐 + BLE 胎压可同时在线，不冲突。

### 315MHz TPMS（原厂协议）

```
  原厂胎压传感器 (315MHz)         主控
  ┌─────────────────┐         ┌──────────────┐
  │ 315MHz 射频发射  │   ═══   │ CC1101 模块   │
  └─────────────────┘  无线   │ ├─ SPI: 接 40-pin   │
                              │ ├─ 弹簧天线 (自带) │
                              │ └─ 3.3V + GND     │
                              └──────────────────┘
```

如果上 CC1101，SPI 接线和 MCP2515 类似（但用不同 CS 引脚）：
| 信号 | Core3566 40-pin |
|------|:---:|
| MOSI | Pin 19（和 CAN 共享 SPI1） |
| MISO | Pin 21 |
| SCLK | Pin 23 |
| CS (CC1101) | Pin 26（GPIO7，和 CAN 的 Pin24 不同） |
| GDO0 (中断) | Pin 22 |
| 天线 | CC1101 模块自带弹簧天线 |

**淘宝关键词**：
| 物料 | 关键词 |
|------|--------|
| BLE 胎压传感器 | "BLE TPMS 内置传感器" |
| CC1101 模块 | "CC1101 315MHz 无线模块 SPI" |

## 六、电源链路

```
车上                                          桌面
┌──────────┐     ┌──────────┐     ┌──────────────┐
│ IKE KL15 │ ──→ │ 12V→5V   │ ──→ │ Core3566     │
│  (Pin 5) │     │ DCDC/车充 │     │ USB-C 口     │
└──────────┘     └──────────┘     └──────────────┘
```

| 阶段 | 方案 |
|------|------|
| **台上测试** | 车载 12V USB-C 充电线（≥2A 输出），插 Core3566 USB-C |
| **正式装车** | 宽压 DCDC 模块（MP4580 / LM2596），输入 6-36V → 稳出 5V |
