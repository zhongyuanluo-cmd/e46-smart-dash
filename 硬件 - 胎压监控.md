
蓝牙同时跑 A2DP 音乐 + BLE 胎压？

**完全可行，而且很轻松。**

```
AP6256 支持的并发：

┌─────────────────────────────────────┐
│  BR/EDR (经典蓝牙)                   │
│  ├── A2DP Sink  (音乐接收)          │
│  ├── AVRCP      (曲目控制)          │
│  └── HFP        (免提通话)          │
│                                     │
│  BLE (低功耗蓝牙)                    │
│  └── 胎压传感器 (被动扫描/连接)      │
│                                     │
│  同时在线，互不干扰                   │
└─────────────────────────────────────┘
```

| 关键点 | 说明 |
|--------|------|
| **协议不同** | A2DP 走 BR/EDR，TPMS 走 BLE，物理层不冲突 |
| **带宽** | 胎压数据每秒几字节，A2DP 328kbps，互不挤占 |
| **内存** | BlueZ 同时管理 BR/EDR + BLE，RK3566 内存毫无压力 |
| **实际场景** | 大部分 TPMS BLE 传感器是 **advertising 广播模式**，持续发 beacon，车机只读不连，更轻量 |

> **唯一注意**：315MHz TPMS 走 CC1101（之前说的方案）和蓝牙 TPMS 是二选一。E46 原厂 TPMS 走 315MHz，后装 TPMS 多用 BLE。你需要根据实际传感器芯片来定。两个都预留也不冲突——SPI 挂 CC1101 + BT BLE 扫描可以共存。
